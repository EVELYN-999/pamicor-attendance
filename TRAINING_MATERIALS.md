## üìñ **Complete User Manual**

### **SECTION 1: GETTING STARTED**

#### **System Requirements**
- Any modern web browser (Chrome, Edge, Firefox, Safari)
- Internet connection (works offline too)
- Touch screen or mouse for signatures

#### **Accessing the System**
- **Web URL**: [Your deployed URL]
- **QR Code**: Scan for instant access
- **Mobile**: Works on all smartphones and tablets

### **SECTION 2: ADMIN FUNCTIONS**

#### **2.1 Creating Meetings**
1. Click **"New Sheet"** in sidebar
2. **Auto-filled**: Date, time based on current session
3. **Required Fields**:
   - Meeting Name (e.g., "Weekly HSE Meeting")
   - Topic/Subject
   - Site & Venue
   - Conducted By

#### **2.2 Managing Attendees**
- **Manual Entry**: Type directly in table
- **Quick Add**: Import from previous meetings
- **Bulk Import**: Upload CSV file
- **Digital Signatures**: Click signature area

#### **2.3 Timeline Management**
- **Morning**: 7:00-8:00 AM (auto-selected before noon)
- **Afternoon**: 1:00-2:00 PM (auto-selected 12-5 PM)
- **Evening**: 6:00-7:00 PM (auto-selected after 5 PM)

#### **2.4 QR Code Generation**
1. Create/select meeting
2. Click **purple "QR" button**
3. **Print QR code** or display on screen
4. Attendees scan to join

#### **2.5 Reports & Export**
- **Individual Meeting**: Click "Export" ‚Üí CSV file
- **Master Report**: Sidebar ‚Üí "Export All Data"
- **Print**: Click "Print" for physical copies

### **SECTION 3: ATTENDEE FUNCTIONS**

#### **3.1 Joining via QR Code**
1. **Open camera** on phone
2. **Point at QR code**
3. **Tap notification** to open link
4. **Fill attendance form**

#### **3.2 Manual Entry**
- **Name**: Full name as per ID
- **Company**: Select from dropdown
- **Job Title**: Select from predefined list
- **Signature**: Draw with finger/stylus

#### **3.3 Late Arrival Tracking**
- System **automatically marks** late arrivals
- **Red "LATE" badge** appears on signatures
- **Timestamp recorded** for all entries

### **SECTION 4: CLOUD FEATURES**

#### **4.1 Connection Status**
- **Green dot**: Online and syncing
- **Yellow dot**: Currently syncing
- **Red dot**: Offline (still works locally)

#### **4.2 Data Backup**
- **Automatic**: Saves every 2 seconds when online
- **Manual**: Click "Save" button anytime
- **Recovery**: Data loads automatically on startup

#### **4.3 Multi-device Access**
- **Same data** across all devices
- **Real-time updates** when multiple users
- **No conflicts** - last change wins

### **SECTION 5: TROUBLESHOOTING**

#### **Common Issues & Solutions**

**Q: QR code doesn't work**
A: Ensure system is deployed online (not local file)

**Q: Signatures not saving**
A: Check internet connection, try manual save

**Q: Can't see old meetings**
A: Check connection status, click refresh button

**Q: Export not working**
A: Ensure browser allows downloads

**Q: Mobile view issues**
A: Use landscape mode for better experience

#### **Contact Support**
- **IT Department**: [Your contact]
- **System Admin**: [Your contact]
- **Emergency**: Use offline mode, sync later

## üèÜ **Best Practices**

### **For Admins:**
1. **Create meetings in advance** for better QR code distribution
2. **Test QR codes** before meetings
3. **Keep backup exports** weekly
4. **Monitor connection status** during meetings

### **For Organizations:**
1. **Print QR codes** on meeting notices
2. **Display QR codes** on screens/projectors
3. **Train key users** as super-users
4. **Regular data backups** recommended
